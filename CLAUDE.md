# Investment Notes - Project Guidelines

## 分析・執筆ワークフロー

- **分析はまずこの会話の中で行い、有意義と判断したものだけをwebにまとめる**
- 流れ：①会話内でデータ取得・分析・議論 → ②採用する内容を合意 → ③.qmdファイルに反映
- Claudeが勝手に分析結果をqmdに書き込まない。必ず会話内での確認を経てから反映する

## 分析ノートブック（.ipynb）の方針

- **探索的な定量分析は `analysis/` フォルダの `.ipynb` に保存する**
- 命名規則：テーマ単位（例：`analysis/copper.ipynb`、`analysis/photonics.ipynb`）
- ユーザーが自分でVS Code等から直接実行できる形にしておく
- カーネルは `investment-notes`（`.venv`を登録済み）を使用する
- **webサイト（.qmd）には煮詰めた結論・チャートのみを載せる。分析の全過程は.ipynbに残す**

## 分析の順序と目的

**分析は必ず以下の順序で行う：**

1. **未来予測（定性・マクロ）を先に行う** — 技術トレンド・市場構造・ボトルネックの変化を考察し、「どういう未来が来るか」の仮説を立てる
2. **その仮説に対して個別企業を評価する** — 「その未来が来たとき、この企業は割安か」という問いで定量指標を使う

現在の株価・時価総額の絶対値から分析を始めない。日々の価格変動に左右されず、構造的な本質を見出すためのモデル。

## 定量分析の表示ルール

- **日本株の財務指標（時価総額・売上等）は必ずJPY→USDに変換して表示する**（yfinanceでUSDJPY=Xレートを取得して換算）
- **時価総額等の絶対値はレンジ表示にする**（市場価格は常に変動するため、正確な値より大まかなレンジが実用的）
  - 例：`$18.3B` ではなく `~$18B` や `$15〜20B` のように表示

## 投資対象ユニバース

- **対象市場：米国上場株（NYSE・NASDAQ）および日本株（東証）のみ**
- 理由：実際に購入できる市場に限定することで、分析を実用的に保つ
- LSE（ロンドン）、ASX（オーストラリア）、EUR市場（ユーロネクスト等）上場銘柄は対象外
  - 例：Glencore（LSE）、Lynas Rare Earths（ASX）、Air Liquide（EPA）は分析対象外
- ADR（米国預託証券）として米国市場に上場している銘柄は対象に含む（例：BHP）

## 投資戦略・資金配分

- **総資金：50万円**
- **リスク資金（10万円）：** 全損OKの高リスク・高リターン枠。小型・投機的銘柄も対象
- **運用資金（40万円）：** 通常運用枠。モートが明確な質の高い銘柄中心
- **目標リターン：2028年までに最低3倍（¥150万円）**
  - 根拠：AI市場規模は2028年までに2〜3倍、2030年までに5倍と予測
  - 上振れ銘柄を選定できれば市場平均を超えるリターンは十分に狙える
- 定量分析では、この2つのバケット（リスク枠・運用枠）への適合度も評価する

## 投資哲学

- **分析対象はその技術・市場でトップレベルの企業を優先する**
- 理由：評価が高い（割高に見える）こと自体が、市場がそのモート（競争優位の堀）を認識している証拠であり、AI特需の恩恵が最も集中するのはそういった企業であるべき
- 割高でも買うか・押し目を待つかの判断軸は「AI特需を織り込んだ成長率に対してバリュエーションが回収可能か」（PEG的視点）
- 定量分析では、単純なPERや絶対値ではなく、成長率・モートの強度・需給構造を合わせて評価する

### AI特需に関する前提（毎回説明不要）

- **AI特需は必ず実現するという確信のもとで投資している**
- NVIDIAですらここからあと4〜5倍になるポテンシャルがあると見ている
- AI特需が実現しなかった場合は資金ゼロになっても構わないという覚悟で臨んでいる
- この前提はすべての銘柄選定・バリュエーション評価の土台であり、毎回確認・説明は不要

## リサーチ方針

- MCP・APIを使うもの（financial-datasets等）以外のリサーチは、必ず外部AIが担当する
- ClaudeがWebサイトを独自に調査・検索することはNG（WebSearchツール等の使用禁止）
- 定性情報（企業概要・競合分析・ニュース等）は外部AIから提供されたものをベースに使用する

## Quartoコード実行

- `_quarto.yml`に`execute: freeze: auto`を設定済み
- コードチャンクはローカルで実行し、結果を`_freeze/`にキャッシュしてgit管理
- GitHub Actions上ではPythonを実行しない（キャッシュを使う）
- ローカルでレンダリング: `quarto render`

## サイト構成方針

- トップページ・Macroセクション：マクロトレンドを前面に出す
- 個別銘柄（stocks/）：深い階層に配置、メインには出さない
- 定性分析：Markdownで記述（.qmdファイル）
- 定量分析：Pythonコードチャンクで記述（同じ.qmdに埋め込み）

## Obsidianノート管理

- **Vault：** `/mnt/c/Users/BlueF/Dropbox/obsidian1/10_Specialty/投資分析 with Claude/`
- **構成：**
  - `_INDEX.md` — 銘柄・テーマへのリンク集（ハブ）
  - `テーマ/` — 投資テーマごとの分析ノート
  - `銘柄/` — 銘柄ごとの分析ノート
  - `会話ログ/` — 整理前の生の会話ダンプ（日付_トピック.md）
- **運用ルール：**
  - `テーマ/`または`銘柄/`に新しいノートを作成したら、必ず`_INDEX.md`の該当セクションにリンクを追加する
  - ノート内では`[[銘柄名]]`や`[[テーマ名]]`でリンクを張り、Graph Viewで関係性が見えるようにする
  - Graph ViewはObsidianアプリのUI機能のため、Claudeからは操作不可。リンクを正しく張ることで自動反映される
  - **知識の更新は弁証法的に行う：** 同じテーマに関する新しいリサーチが来たら、`会話ログ/`にその回の議論をダンプしつつ、`テーマ/`・`銘柄/`ノートは現時点での統合理解に上書き更新する。過去の経緯は会話ログに残すため、テーマ/銘柄ノートは常に「今の最善理解」だけを反映した状態に保つ

## セッション間の記憶管理

- **会話ログはObsidianの `会話ログ/` に保存する**（Claudeのmemoryシステムは使わない）
- セッション終了時または区切りのよいタイミングで、`会話ログ/日付_トピック.md` としてダンプする
- 命名例：`2026-05-19_AIデータセンターPhase2電力分析.md`
- 次セッション開始時にユーザーがそのファイルを貼り付けることで文脈を引き継ぐ

## デプロイフロー

1. ローカルで分析・執筆
2. `quarto render`でビルド（コードチャンクが実行され`_freeze/`に保存）
3. `git add . && git commit && git push origin main`
4. GitHub Actionsが`gh-pages`ブランチを更新
5. GitHub Pagesに自動反映（数分）
